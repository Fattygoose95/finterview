## 🎯 Why finterview?

Finance interviews are high‑stakes, time‑sensitive, and demand precise answers. Most resources are either too verbose or lack real‑world context.

**finterview** delivers:
- **Concise, actionable answers** – bullet‑point summaries that capture the core of each question
- **Professional design** – a clean, finance‑focused UI that feels like a Bloomberg Terminal
- **Smart filtering** – practice by frequency, difficulty, role, or industry
- **Zero‑fluff** – no long essays, no filler content, just what you need to perform

Built by a finance professional with AI‑assisted development, the platform reflects how top candidates actually prepare.

## 🚀 Core Features

### 1. **High‑Frequency Question Mode**
- Questions ranked by real‑interview appearance
- Clean list layout with instant answer toggle
- Search and filter by category, difficulty, role

### 2. **Random Practice Mode**
- Multi‑dimensional filtering (stars, type, difficulty, tags)
- Multiple sorting options (random, frequency, difficulty)
- Toggle between concise and detailed answers

### 3. **Professional Finance Design System**
- Bloomberg‑inspired dark‑blue color scheme
- Fully responsive, mobile‑optimized layout
- Consistent component variables and spacing

### 4. **Concise Answer Engine**
- 79+ questions converted to bullet‑point summaries
- Average answer length reduced by **71.4%** (895 → 219 characters)
- Preserves full original answers for depth when needed

### 5. **Industry‑Specific Practice**
- Filter by investment banking, asset management, quantitative finance, sales & trading
- Role‑based question sets that mirror actual interview flows

## 🏗️ Technical Architecture

```text
finterview/
├── core/                          # Source code
│   ├── css/                      # Design system + components
│   ├── js/                       # Data loader, modes, main logic
│   ├── *.html                    # All pages (high‑frequency, random, etc.)
│   └── questions.js              # Complete question dataset (79+)
├── generated/                    # Generated/processed files
├── backups/                      # Versioned backups
└── docs/                         # Project documentation
```

**Key technical decisions:**
- **Vanilla HTML/CSS/JS** – no frameworks for maximum speed and simplicity
- **GitHub Pages** – zero‑cost hosting with automatic deployment
- **Modular JavaScript** – separate data‑loading, filtering, and UI logic
- **Design‑system approach** – CSS custom properties for consistent theming
